# MyReps.info

A nonpartisan, open reference site for every elected official in the United States — local, state, and federal.

**Live site: [myreps-info.vercel.app](https://myreps-info.vercel.app)**

---

## What It Does

Enter any U.S. ZIP code to find every politician representing you at every level of government. Each politician's page includes:

- Full voting record on every bill they've voted on while in office
- Legislation they've sponsored or co-sponsored
- Plain-English summaries of every bill (sourced from the Congressional Research Service, or generated by Claude when no CRS summary exists)
- Contact information

---

## Data Sources

| Source | What it provides |
|--------|-----------------|
| [OpenStates](https://openstates.org) | ZIP code lookups, state legislative data for all 50 states |
| [Congress.gov API](https://api.congress.gov) | Federal members, bills, votes, and CRS summaries |
| [Nominatim / OpenStreetMap](https://nominatim.openstreetmap.org) | ZIP code geocoding |
| [Claude (Anthropic)](https://anthropic.com) | Plain-English bill summaries when no CRS summary exists |

Data is refreshed automatically every Sunday via a GitHub Actions workflow.

## Editorial Policy

MyReps.info is strictly nonpartisan. The site presents official voting records and factual biographical information only — no editorial framing, commentary, or partisan perspective.

**This is a read-only reference site. Pull requests are not accepted.**

All data originates from official government APIs. If you believe a factual error exists in a politician's record or a bill summary, the error is almost certainly in the upstream source. Please report it to [Congress.gov](https://api.congress.gov) or [OpenStates](https://openstates.org) directly.

Bill summaries generated by Claude are clearly labeled on each bill page. The prompt used to generate them is designed to be strictly factual and nonpartisan:

> *"You are a nonpartisan legislative assistant. Write a clear, factual, plain-English summary of the following bill. Do not include political opinions or partisan framing. Cover: what the bill proposes, who it affects, and the key provisions."*

## Architecture

MyReps.info uses an **LLM Wiki** pattern: a scheduled pipeline pulls from government APIs, writes structured Markdown files for every politician and bill, and the site is generated statically from those files. Pages load instantly and the site works even if upstream APIs are temporarily unavailable.

```
Political APIs → Ingestion pipeline (weekly) → wiki/ Markdown files → Next.js static site → Vercel
```

### Stack

- **Frontend:** Next.js 15, Tailwind CSS
- **Hosting:** Vercel (auto-deploys on push)
- **Data pipeline:** TypeScript script, runs via GitHub Actions every Sunday
- **APIs:** OpenStates, Congress.gov, Nominatim, Anthropic

## Running Locally

```bash
git clone https://github.com/tashacerny/myreps-info.git
cd myreps-info
npm install
npm run dev
```

The site runs in demo mode locally (no API keys needed). To run the full ingestion pipeline, copy `.env.example` to `.env` and add your API keys.
